Assignment 12:
Pig Dice: A JavaScript Game

Write a JavaScript implementation of the game Pig Dice.

Due: TBD

Objective

  • Learn how to use closures to implement objects in JavaScript
  • Learn to write unit tests in JavaScript.
  • Obtain additional practice using JavaScript.

Rules of the Game

The goal is to be the first player to reach the target number of points (e.g., 100).

Players take turns. On his turn, a player may roll the die as often as he likes and calculates the total value of all the rolls. The total for a given turn is called the point. When he chooses to pass his turn (i.e., stop rolling), he adds the point to his current score. However, if at any time a player rolls a 1, his turn ends immediately, and he does not add any points to his total.

Details

Implement the game Pig Dice in JavaScript and implement appropriate unit tests.

  1. Your design should have at least two separate MVC triads:
    1. The first triad will be in charge of gathering the game's configuration information: player names and the target score. Ignore empty player fields (as opposed to creating players with no name.)
    2. The second triad will be in charge of running the game itself.
    The first triad will place a handler on the configuration form. That handler will create the second triad, hide the configuration window, and show the game board.
  2. Your triads should use good MVC or MVP design. In other words, the game logic should be separate from the code that interacts with your web page.
    • The "view" code interacts with your web page. It returns information from your page (e.g., reads forms) and updates your page (e.g., updates the score).
    • The "model" code contains the game state and logic (e.g., keeps track of scores and whose turn it is).
    • The controller or presenter "glues" the model and view together. In general, your event handlers are implemented here.
    (Your game configuration triad doesn't need a model --- just a view and controller.)
  3. In addition to the triads, you should have separate GameDie and Player classes. These classes need not be triads.
  4. Each "model" class (including GameDie and Player) should have its own test suite. For this project, writing unit tests for the controller/presenter is optional.

Starter Code

The git repository listed below contains extensive starter code. Cloning this repository will give you a directory structure and shells for all the main classes in the project. The directory structure will show you one way of organizing a large JavaScript project. The "shells" for the files will save you the time and effort of setting up all the MVC triads and JavaScript objects. The pigDiceGame.html shell will save you the trouble of loading all the different Java.

You are not required to use this starter code, but (1) it will save you a lot of time, and (2) it will help ensure you are using a good MVC/MVP design.

I suggest you begin with the PigGameConfig traid, because I've included the most starter code there.

Testing

You must write Jasmine tests for each model class. To add new tests:

  • Add a new file to jasmine/spec.
  • Add a <script> tag to jasmine/SpecRunner.html to include the new test.

  • 10 points: Visual design

You will lose points if your code is not reasonably well formatted and documented.
